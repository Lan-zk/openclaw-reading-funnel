# Docs

This directory is reserved for longer-form project documentation that sits beside
the planning pack.

Use these subdirectories when implementation begins to generate stable docs that
describe what actually shipped:

- `product/` for longer-form product rationale and user-facing concepts
- `architecture/` for diagrams, contracts, and technical notes derived from code
- `operations/` for runbooks, validation notes, and incident guidance

Until implementation artifacts exist, the planning source of truth remains in
`planning/openclaw-storypack-inbox/`.
